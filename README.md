# Dashboard Project

## Step 1: Set Up and Planning

-   Set up your **HTML and CSS files** with some dummy content to make sure everything is linked correctly.
-   Use .png-file at the root of the project as the template of a final result.
-   Use percentages, fractional or relative values for the structure and sizes definition. Certain specific values must be used only when there's no other option.
-   Avoid mess and duplications.

## Step 2: Layout

-   Start by writing out the HTML elements for the **sidebar**, **header**, and **main-content** containers.
-   In your CSS file, apply **Grid properties** until you have this basic layout built.

## Step 3: Nesting

Taking it one section at a time, begin nesting child elements under the parent elements in the HTML. Remember that you can keep making grid containers within grid containers.

-   **Sidebar** — use more grids to lay out the navigation and branding sections.
-   **Header** — use more grids to lay out the search bar, user info, and buttons.
-   **Main Content** — use more grids to lay out the projects, announcements, and trending items.
-   Fill out some **dummy content** and **placeholder images** so you can position all of your grid items.

## Step 4: Gather Assets

Once your grid layout is complete, you can either recreate the dashboard example or style your own design.

-   All icons and more can be downloaded as SVGs from [Material Design Icons](https://materialdesignicons.com/).
-   Choose your own **fonts**! The design example uses _Roboto_, available via [Google Fonts](https://fonts.google.com/). _(Review the More Text Styles Lesson for information on including external fonts in your projects.)_

## Step 5: Tips

-   Apply **background colors or borders** to your containers while building the layout to help visualize your grid.
-   It's up to you whether to use `px`, `fr` units, or both for your grid tracks.
-   This project does **not** have to be responsive, but you can optionally expand or shrink the project cards section when resizing the browser window.
-   You don't have to make a **pixel-perfect match** with the design example — consider this an opportunity to practice your CSS skills with your own designs.
